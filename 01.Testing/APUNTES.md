# Testing Pro

## Introducción

Los **tests automáticos** son una serie de pruebas que se realizan para verificar que un programa funcione correctamente y que no se rompe cuando se realizan modificaciones. Se realizan de manera **automática** para ejecutarlas en cualquier momento, ya que son **fundamentales** del desarrollo de nuestro software.

## Tests usados en tiempo de Desarrollo

- Tests **unitarios**
- Tests de **aceptación**
- Tests de **integración** (o de componentes)
- Tests **end-to-end**

## Tests usados en procesos posteriores al de Desarrollo

- Tests de **regresión**: Se usan en las actualizaciones o mejoras en la aplicación.
- Tests de **carga**: Analizan el comportamiento del software cuando varía la carga de datos.
- Tests de **rendimiento**: Revisan como responde la aplicación ante un determinado flujo de tráfico de usuarios y de información. Test de alto nivel.

## ¿Quién se encarga de realizar el testing?

Al ser parte de la comprobación de la calidad y la eficacia de la aplicación, es el **equipo de calidad (Q&A)** o un equipo de operaciones el encargado de esta tarea. Pero existen test más específicos para los desarrolladores.
